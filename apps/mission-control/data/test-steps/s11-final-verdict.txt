### 3. Security Audit: **FAIL — 4 Critical, 10 High unresolved**

| Severity | Count |
|---|---|
| Critical | 4 |
| High | 10 |

No critical or high findings have been remediated. The four critical findings are individually release-blocking:

1. **A01 — No location scoping** on `GET/PATCH /clients/:id`: any authenticated staff JWT from any location can read or modify any client in the system — a multi-tenant data breach vector
2. **A01 — No location scoping** on `GET /redemptions/:code`: cross-location redemption data exposure
3. **A03 — DATE_TRUNC injection surface**: the fix requires safe interpolation with an allowlist; the current code both crashes at runtime and would be an injection vector if naively patched
4. **A07 — PwaJwtGuard ignores Authorization header**: already a P0 functional bug, also a security gap — the guard's incomplete token extraction logic means the redemption endpoint is simultaneously broken and improperly guarded

The `RolesGuard` bypass (Finding 3, High) is also near-critical: `Reflect.defineMetadata` on `descriptor.value` vs. `SetMetadata` means **all `@Roles()` decorators are silently no-ops**, allowing any authenticated staff member to reach owner-only endpoints. Combined with the missing location-scoping, the authorization model is effectively absent.

The hardcoded secrets in `docker-compose.yml` (Finding 12) and the absent JWT secret validation (Finding 6) represent a credential hygiene failure that must be resolved before any staging or production deployment.

## REASON

The product cannot be released in its current state. Five P0 technical defects cause complete functional failures in the three core value-delivery mechanisms (SMS, PWA, redemption), four critical security findings expose multi-tenant client data and bypass the authorization model entirely, and 11 of 24 acceptance criteria fail outright — including all seven P0 ACs that define the product's primary user experience. The underlying architecture is sound and the majority of the codebase is well-constructed; the failures are concentrated in a specific set of integration-layer bugs (import semantics, guard logic, handler assignment order, missing scheduler, truncated static assets) that are individually tractable but collectively represent a release-blocking remediation sprint.

## UNRESOLVED ISSUES

### P0 — Ship Blockers (must fix before any release)

1. **`import * as Twilio`** without `esModuleInterop: true` → `TypeError: Twilio is not a function` at runtime; fix in `sms.service.ts` and `worker/src/sms/twilio.service.ts`
2. **`PwaJwtGuard` ignores `Authorization` header** → all `POST /redemptions/request` calls return 401; guard must extract token from Bearer header for POST endpoints
3. **`ReferralBonusHandler` referrer/referee swap** → sort-based deadlock prevention destroys assignment logic; fix by querying each ID explicitly rather than relying on sort position
4. **`DATE_TRUNC($1, ...)` parameterization** → PostgreSQL runtime error on all analytics queries; fix with validated safe interpolation from closed enum allowlist
5. **`styles/globals.css` truncated mid-declaration** → all PWA styling broken; file must be completed; `pwa.css` must be created
6. **`RolesGuard` / `Roles` decorator broken** → `Reflect.defineMetadata(ROLES_KEY, roles, descriptor.value)` never matches `Reflector.get(ROLES_KEY, context.getHandler())`; replace with `SetMetadata`
7. **No location scoping on `GET/PATCH /clients/:id`** → cross-tenant client data accessible to any authenticated user
8. **No birthday/anniversary scheduler** → AC-19 milestone automation is entirely absent; a cron job or scheduled Lambda must be implemented
9. **Referral bonus never triggered by checkout flow** → `CheckoutAwardHandler` does not detect referred clients or enqueue `REFERRAL_BONUS` events; trigger logic is missing
10. **`acknowledgeEventResult` race condition** → balance recompute runs outside transaction without `FOR UPDATE` lock; concurrent events produce stale tier state

### P1 — Release-Critical (must fix before GA)

11. **`dynamic UPDATE idx` bug in `milestones.repository.ts`** → partial updates produce wrong WHERE clause parameters
12. **Referral code exhaustion** → after 10 failed uniqueness attempts, non-unique code is used, causing DB constraint violation
13. **`POST /redemptions/:code/apply` returns 404 for all invalid states** → contract requires 409 (used) and 410 (expired); violates API contract
14. **`gdpr-purge-job.sh` truncated** → script fails with syntax error on execution; GDPR compliance mechanism is broken
15. **`branding.repository.ts` upsert with empty body on existing record** → produces invalid SQL `DO UPDATE SET , updated_at = NOW()`
16. **`const crypto = require('crypto')` and `const jwt = require('jsonwebtoken')`** inside ES module methods → replace with top-level `import` statements
17. **`internal.service.ts` trusts worker-supplied `new_tier_id`** without validating tier belongs to client's location (security Finding 16)

### P1 — Security (release-critical)

18. **No location scoping on `GET /redemptions/:code`** → cross-tenant redemption data exposure
19. **No location scoping on transaction history endpoint**
20. **JWT secret not validated for presence/minimum length** → empty string secret allows token forgery
21. **`selectForUpdate` table name not allowlisted** → future SQL injection surface

23. **Hardcoded secrets in `docker-compose.yml`** → must be externalized before any non-local deployment
24. **`logo_url`/`image_url` accept non-HTTPS and private IP ranges** → stored SSRF/XSS vector
