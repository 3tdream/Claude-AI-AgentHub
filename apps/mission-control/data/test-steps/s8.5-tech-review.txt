### P0 ISSUE AUDIT

**P0-1: `src/auth/roles.guard.ts` — Roles decorator bypasses NestJS Reflector**
Confirmed in code: `Reflect.defineMetadata(ROLES_KEY, roles, descriptor.value)` is used instead of NestJS `SetMetadata`. `this.reflector.get<string[]>(ROLES_KEY, context.getHandler())` will return `undefined` because `context.getHandler()` returns the method reference, not `descriptor.value` (they are the same object in most cases, but the NestJS Reflector pipeline uses its own metadata key namespace via `SetMetadata`). **Result: all role-protected routes pass through unguarded in production.** ✅ Confirmed P0.

**P0-2: `src/modules/redemptions/redemptions.controller.ts` — PwaJwtGuard only reads query/path param, not Authorization header**
Confirmed in `pwa-jwt.guard.ts`: `const token = request.query.t || request.params.token` — no Authorization header read. `redemptionsApi.request` in `lib/loyalty-api.ts` sends `Authorization: Bearer <pwaToken>` via the `token` field in `apiFetch`. The `POST /redemptions/request` body contains no `t` query param. **Result: 100% of PWA redemption requests return 401.** ✅ Confirmed P0.

**P0-3: `worker/src/handlers/referral-bonus.handler.ts` — Referrer/referee identity swap**
Confirmed: `ids.sort()` reorders UUIDs, then `referrerRow = query(ids[0])` and `refereeRow = query(ids[1])`. The code then uses `referrer_client_id` and `referee_client_id` from `payload` to update balances — but the `referrer` and `referee` variables are bound to `ids[0]` and `ids[1]` which may be in either order. The balance update queries use `referrer_client_id` and `referee_client_id` directly (correct), but the transaction descriptions and `referrerNewBalance`/`refereeNewBalance` are computed from the sorted-order rows. **For ~50% of pairs, bonus amounts are swapped.** ✅ Confirmed P0.

**P0-4: `src/modules/notifications/sms.service.ts` — `import * as Twilio from 'twilio'` without esModuleInterop**
Confirmed in `tsconfig.json`: `esModuleInterop` is **absent** (not set to true). `import * as Twilio from 'twilio'` then calling `Twilio(...)` will throw `TypeError: Twilio is not a function` at runtime. Same issue in `worker/src/sms/twilio.service.ts`. **Result: all SMS sending fails at startup/first call.** ✅ Confirmed P0.

**P0-5: `src/modules/analytics/analytics.repository.ts` — `DATE_TRUNC($1, ...)` parameterized field**
Confirmed: `DATE_TRUNC($1, created_at)` with `[truncFn, locationId, dateFrom, dateTo]`. PostgreSQL does not accept a parameter placeholder as the `date_trunc` field argument. **Result: all analytics points-activity queries throw PostgreSQL runtime error.** ✅ Confirmed P0.

**P0-6: `styles/globals.css` — File truncated mid-declaration**
Confirmed: the CSS file ends at `--font-size-` with no closing value, semicolon, or `}`. **Result: CSS parse error breaks all styling.** ✅ Confirmed P0.
