### 2. CI/CD PIPELINE

**`deploy-api.yml`** — 8-stage pipeline:
```
lint → test → security → build-push → migrate → deploy → smoke-test → rollback
```

**`deploy-worker.yml`** — 7-stage pipeline (independent from API):
```
lint → test → security → build-push → deploy → verify → rollback
```

**`pr-checks.yml`** — Fast PR gate (no deploy):
```
lint-typecheck → test → security-audit (gitleaks)
```

Key CI/CD decisions:
- **OIDC auth** — no long-lived AWS keys in GitHub secrets
- **Coverage gate** — 80% API / 75% worker minimum enforced
- **Trivy scanning** — blocks on CRITICAL/HIGH CVEs
- **Migration-first** — DB migrations run as ECS one-off task before service deploy
- **Smoke tests** — auth guard, health, and 5xx detection post-deploy

### 3. DEPLOYMENT CONFIG

### 4. ROLLBACK PLAN

**Automated rollback** (triggered on smoke test failure):
1. Pre-deploy: current task definition ARN snapshotted → stored in **SSM Parameter Store** (`/beauty-crm/loyalty/api/last-stable-task-def`)
2. On failure: `aws ecs update-service --task-definition {SSM_VALUE} --force-new-deployment`
3. Waits for `services-stable` confirmation before marking rollback complete
4. Separate SSM keys for API and Worker (`/beauty-crm/loyalty/worker/last-stable-task-def`)

**JWT Secret rotation** (`scripts/rotate-jwt-secret.sh`) — 5-step safe rotation:
```
generate → deploy-dual → verify-drain → promote → deploy-final
```
Addresses ADR-001 consequence: dual-secret grace period prevents in-flight SQS message invalidation during rotation.

**Database rollback**: Migrations are tracked in `_schema_migrations` with SHA256 checksums — mutated migrations are detected and blocked. RDS 14-day PITR available for data recovery.
