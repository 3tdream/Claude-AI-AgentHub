# Complete File Plan — Beauty CRM Loyalty Module

## Backend-Agent Files

### Database Migrations
- `migrations/001_create_organizations.sql` — creates `Organization` table with PK and indexes
- `migrations/002_create_locations.sql` — creates `Location` table with FK to Organization, indexes
- `migrations/003_create_loyalty_config.sql` — creates `LoyaltyConfig` table with unique constraint on `location_id`
- `migrations/004_create_loyalty_tiers.sql` — creates `LoyaltyTier` table with composite unique on `(loyalty_config_id, order)`, cascade delete
- `migrations/005_create_category_earn_rates.sql` — creates `CategoryEarnRate` table with composite unique on `(loyalty_config_id, category_id)`
- `migrations/006_create_rewards.sql` — creates `Reward` table with soft-delete `deleted_at`, indexes on `(location_id, active)`
- `migrations/007_create_loyalty_clients.sql` — creates `LoyaltyClient` table with GDPR fields, self-ref FK, unique referral_code, composite unique on `(location_id, crm_client_id)`
- `migrations/008_create_point_transactions.sql` — creates `PointTransaction` append-only ledger with unique `idempotency_key`, composite index on `(loyalty_client_id, created_at DESC)`
- `migrations/009_create_redemption_codes.sql` — creates `RedemptionCode` table with unique `code`, partial unique index on `(loyalty_client_id, reward_id)` where `status = 'pending'`
- `migrations/010_create_referrals.sql` — creates `Referral` table with unique `referee_client_id`, snapshot bonus fields
- `migrations/011_create_milestones.sql` — creates `Milestone` table with composite unique on `(location_id, type)`
- `migrations/012_create_branding.sql` — creates `Branding` table with unique constraint on `location_id`
- `migrations/013_create_notification_logs.sql` — creates `NotificationLog` append-only table with indexes on `(location_id, sent_at DESC)` and `(location_id, status)`
- `migrations/014_seed_milestone_types.sql` — seeds `BIRTHDAY`, `FIRST_VISIT`, `PROGRAM_ANNIVERSARY` milestone rows per location trigger

### Points Engine Worker (Separate ECS Service)
- `worker/src/main.ts` — NestJS worker bootstrap, SQS consumer initialization

- `worker/src/queue/sqs.producer.ts` — enqueue helper for award and milestone events to `POINTS_ENGINE_EVENT_QUEUE`
- `worker/src/handlers/checkout-award.handler.ts` — processes `CHECKOUT_AWARD` events: calculates points, writes ledger, calls back API
- `worker/src/handlers/milestone-award.handler.ts` — processes `MILESTONE_BONUS` events: validates type, awards points, triggers SMS
- `worker/src/handlers/referral-bonus.handler.ts` — processes `REFERRAL_BONUS` events: awards referrer and referee points atomically
- `worker/src/handlers/tier-recalculation.handler.ts` — evaluates new tier after balance change, updates `current_tier_id`
- `worker/src/idempotency/redis.service.ts` — Upstash Redis client, 15-minute TTL idempotency key check and set
- `worker/src/sms/twilio.service.ts` — Twilio SMS dispatch, opt-in guard, monthly cap check, writes `NotificationLog`

- `src/auth/roles.guard.ts` — enforces owner-only vs owner-or-staff role restrictions per endpoint
- `src/auth/auth.module.ts` — NestJS auth module wiring all guards and strategies

## DevOps-Agent Files

- `.env.example` — all required env vars: `DATABASE_URL`, `REDIS_URL`, `AWS_SQS_QUEUE_URL`, `AWS_SQS_DLQ_URL`, `AWS_REGION`, `LOYALTY_JWT_SECRET`, `LOYALTY_LINK_EXPIRY_HOURS`, `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_FROM_NUMBER`, `SMS_MONTHLY_LIMIT_PER_BUSINESS`, `REFERRAL_CODE_LENGTH`, `GDPR_DATA_RETENTION_DAYS`, `INTERNAL_HMAC_SECRET`, `POINTS_ENGINE_EVENT_QUEUE`
- `Dockerfile` — multi-stage build for CRM API NestJS service (node:20-alpine, production target)
- `worker/Dockerfile` — separate multi-stage build for SQS worker ECS Fargate task
- `docker-compose.yml` — local dev stack: postgres:15, redis, localstack (SQS emulation), api, worker services
- `infra/ecs-task-definition.json` — ECS Fargate task definition for worker service with env var injection
- `infra/sqs-queue.tf` — Terraform: `POINTS_ENGINE_EVENT_QUEUE` FIFO queue, DLQ, redrive policy, alert on DLQ depth
- `infra/iam-worker-role.tf` — Terraform: IAM role for ECS worker with SQS consume and DLQ send permissions
- `infra/rds-postgres.tf` — Terraform: PostgreSQL 15 RDS instance with parameter group and backup config
- `infra/elasticache-redis.tf` — Terraform: Upstash Redis or ElastiCache cluster for idempotency key TTL cache
- `.github/workflows/deploy-api.yml` — CI/CD pipeline: lint, test, build, push ECR image, deploy ECS API service
- `.github/workflows/deploy-worker.yml` — CI/CD pipeline: build and deploy worker ECS Fargate task separately
- `scripts/run-migrations.sh` — migration runner script executed as ECS one-off task before deploy
- `scripts/gdpr-purge-job.sh` — scheduled Lambda/cron script: hard-purges `LoyaltyClient` records past `retention_until`
- `scripts/birthday-cron.sh` — daily cron: queries clients with today's birthday, calls POST `/internal/milestones/trigger`
- `scripts/anniversary-cron.sh` — daily cron: queries clients at program anniversary date, triggers milestone events
- `scripts/expire-redemption-codes.sh` — periodic job: sweeps `RedemptionCode` rows past `expires_at`, sets status to `expired`
