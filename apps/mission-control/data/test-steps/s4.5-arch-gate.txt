## CHECK 3 — CYBER FINDINGS RESOLUTION

### Finding 1 — PWA JWT Token in URL Path (Critical)

**Prescribed Fix:** Move token to query parameter as immediate mitigation; implement token-exchange flow (opaque short-lived code → session cookie); enforce short `LOYALTY_LINK_EXPIRY_HOURS`; add `Referrer-Policy: no-referrer`.

**Architecture Review:**

| Fix Component | Architecture Status | Assessment |
|---|---|---|
| Token in URL path vs. query param | GET `/pwa/client/:token` — token remains in **path parameter** across API contract, file plan, and ERD | ❌ Not addressed |
| Token-exchange flow (opaque code → session cookie) | No token-exchange endpoint exists in the 32-endpoint contract; `pwa-jwt.guard.ts` validates long-lived JWT directly | ❌ Not addressed |
| Short `LOYALTY_LINK_EXPIRY_HOURS` recommendation | Env var exists; no minimum value enforced in contract or guard | ⚠️ Partial |
| `Referrer-Policy: no-referrer` header | Not present in any response header specification in API contract or file plan | ❌ Not addressed |

**Finding 1 STATUS: ❌ UNRESOLVED** — The path-parameter token pattern is structurally unchanged in all architecture documents. The cyber recommendation was acknowledged but not incorporated into the API contract, guard implementation, or file plan.

---

### Finding 2 — Internal HMAC Endpoints Lack Network Isolation (Critical)

**Prescribed Fix:** VPC-internal ALB or security group isolation for `/internal/*` endpoints; refactor `event-result` to discard `new_balance` and recompute from ledger.

**Architecture Review:**

| Fix Component | Architecture Status | Assessment |
|---|---|---|
| Network-layer isolation (private VPC ALB / security group) | `infra/ecs-task-definition.json` and `infra/iam-worker-role.tf` exist but contain no network isolation specification; no private ALB or security group rule defined in Terraform files | ❌ Not addressed |
| Discard `new_balance` from event-result payload | `POST /internal/points-engine/event-result` request body still includes `new_balance: number` as a field; `internal.service.ts` description says "tier recalculation dispatch" but does not specify balance recomputation | ❌ Not addressed |
| HMAC guard remains sole protection | `src/auth/hmac.guard.ts` is the only auth mechanism listed; no network boundary enforced | ❌ Not addressed |

**Finding 2 STATUS: ❌ UNRESOLVED** — Both sub-findings (network isolation and balance source-of-trust) are unaddressed in the architecture. The `new_balance` field remains in the event-result contract as an attacker-controllable balance override.

---

### Finding 3 — Redemption Code Enumeration and Timing Oracle (Critical)

**Prescribed Fix:** 128-bit cryptographic randomness for codes; normalize invalid-code responses to single 404; enforce `location_id` scoping on apply; reduce rate limit to 10/min per staff ID.

**Architecture Review:**

| Fix Component | Architecture Status | Assessment |
|---|---|---|
| 128-bit cryptographic randomness | `referral-code.generator.ts` exists for referral codes but **no equivalent secure generator** is specified for `RedemptionCode.code`; `max 20 chars` constraint with no entropy floor documented | ❌ Not addressed |
| Normalize 404/409/410 to single 404 | API contract still documents distinct 404/409/410 responses for apply endpoint | ❌ Not addressed |
| `location_id` validation on apply | API contract notes for apply endpoint do not mention location scoping; `redemptions.service.ts` description does not specify this check | ❌ Not addressed |
| Rate limit 10/min per staff ID | API contract still shows 60/min for apply endpoint; no per-staff-id tracking specified | ❌ Not addressed |

**Finding 3 STATUS: ❌ UNRESOLVED** — All four sub-findings remain unaddressed across API contract, ERD, and file plan.

## ARCHITECT REDESIGN BRIEF

The architecture is structurally sound and functionally comprehensive — 10/10 user stories are covered, the data model is well-designed, and the file plan is largely complete. The failure is isolated to **security architecture** (all 3 Critical cyber findings passed through without incorporation) and **2 minor functional gaps** (AC-12 retry throttle, missing redemption code generator). This is a focused remediation, not a full redesign.

### Required Changes for Gate Re-submission

**Priority 1 — API Contract Revisions (Domain 8 + Domain 12 + Domain 5):**
1. Change GET `/pwa/client/:token` from path param to query param (`/pwa/client?t=<token>`) and add a token-exchange endpoint (`POST /pwa/auth/exchange`) that converts a single-use opaque SMS code to a short-lived session cookie
2. Remove `new_balance` from `POST /internal/points-engine/event-result` request body; document server-side ledger recomputation as authoritative
3. Normalize `POST /loyalty/redemptions/:code/apply` error responses to single `404` for all invalid-code states; reduce rate limit to 10/min with per-staff-id tracking; add explicit `location_id` validation note

**Priority 2 — File Plan Additions:**
1. Add `src/common/utils/redemption-code.generator.ts` using `crypto.randomBytes()` with minimum 128-bit entropy
2. Add `infra/private-alb.tf` with security group rules restricting `/internal/*` to worker ECS task security group only
3. Add `Referrer-Policy: no-referrer` to PWA response headers in `pwa.controller.ts`
4. Add retry throttle specification to `worker/src/sms/twilio.service.ts` (max 1 retry per client per notification event within 1-hour window)

**Priority 3 — Minor Gaps (Recommended for same revision pass):**
- G7 through G15 as listed above
